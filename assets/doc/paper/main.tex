\abstract{
\textbf{Summary:} Batch effects remain a major obstacle to reproducible microbiome analysis and cross-study integration. Practical batch correction remains nontrivial because existing methods differ in assumptions, required inputs and evaluation outputs. Here, we present mBatchNet, an interactive web server that \rev{integrates established methods for} microbiome batch-effect diagnosis, correction and post-correction assessment within a single workflow. The server integrates representative microbiome-specific and general-purpose correction methods, standardized pre- and post-correction visualizations and statistical diagnostics, downloadable corrected matrices and run logs, and a built-in example dataset with step-by-step guidance. In a 16S rRNA anaerobic digestion case study, mBatchNet revealed method-dependent differences in batch attenuation and phenotype preservation, highlighting its utility for comparing correction strategies.\\
\textbf{Availability and implementation:} mBatchNet is freely available without login at \rev{\href{https://mbatchnet.com/}{https://mbatchnet.com/}}. Source code \rev{is} available at \href{https://github.com/gilmore307/mBatchNet}{https://github.com/gilmore307/mBatchNet}. The server is implemented with a Python/Dash front end and coordinated Python/R back-end analysis scripts.\\

\section{Introduction}
High-throughput microbiome studies are increasingly integrated across cohorts, centers and time points, but abundance profiles are highly susceptible to batch effects arising from sample handling, DNA extraction, library preparation, sequencing runs and preprocessing pipelines~\citep{Costea2017FecalProcessing,McLaren2019MetagenomicBias,Sinha2017MBQC}. Because microbiome tables are sparse, compositional and often zero-inflated, technical variation can obscure biological structure and confound downstream association analysis, especially when batch and phenotype are only partially separable~\citep{Ling2022ConQuR}. \rev{Sparsity and zero inflation make feature detection uneven across samples, while compositional scaling means that technical shifts in one set of features can affect the relative abundance of others.}

Recent microbiome-oriented \rev{batch-effect correction} methods, including ConQuR \citep{Ling2022ConQuR}, MMUPHin \citep{Ma2022MMUPHin}, PLSDA-batch \citep{Wang2023PLSDABatch}, DEBIAS-M \citep{Austin2025DEBIASM} and MetaDICT \citep{Yuan2025MetaDICT}, offer complementary modeling assumptions and strengths. However, practical correction and evaluation remains challenging because tools differ in input requirements, \rev{method parameters} and output formats. Existing resources partly address this need. BatchServer \citep{Zhu2021BatchServer} provides a general omics web interface centered on ComBat-based correction and a limited diagnostic set, whereas MBECS packages multiple microbiome correction and evaluation routines for R users~\citep{Olbrich2023MBECS}. To our knowledge, a browser-based platform that specifically targets microbiome abundance data while exposing recent microbiome-specific correction methods and harmonized pre- and post-correction diagnostics has been lacking.

We developed mBatchNet to fill this gap. The server combines data upload, study-design inspection, execution of representative correction methods, standardized benchmarking of correction outcomes, and export of corrected matrices and figures within a reproducible session-based workflow. The public deployment includes loadable example data, help pages and a tutorial so that users can reproduce the complete analysis from data upload to result interpretation.

\section{Features and implementation}
mBatchNet uses a Python/Dash web interface with Bootstrap components, Dash AG Grid tables and client-side callbacks for responsive navigation; computational routines are executed through coordinated Python and R back-end scripts. Each analysis run is session scoped. The server records user-selected \rev{metadata variables and method parameters}, writes intermediate and final results to an isolated working directory, streams logs to the browser, and packages outputs for download as a ZIP archive.

The upload module accepts a \rev{valid} count or abundance table together with matching sample metadata in CSV format. Consistent with the public code and example data, the feature table is organized with \rev{samples as rows and features as columns}. The metadata must contain \rev{a batch variable and a target variable} and may include additional covariates. After column mapping, mBatchNet summarizes the study design with preview tables and a batch-versus-target mosaic plot to flag potential imbalance.

The correction module implements microbiome-oriented methods, including ConQuR \citep{Ling2022ConQuR}, MMUPHin \citep{Ma2022MMUPHin}, PLSDA-batch \citep{Wang2023PLSDABatch}, DEBIAS-M \citep{Austin2025DEBIASM}, and MetaDICT \citep{Yuan2025MetaDICT}, together with commonly used adjustment frameworks adapted from gene-expression studies, namely ComBat \citep{Johnson2007ComBat}, limma \citep{Ritchie2015limma}, ComBat-seq \citep{Zhang2020ComBatSeq}, and FAbatch \citep{Hornung2016FAbatch}.
Users can run one or more batch-effect correction methods while specifying method-specific parameters. \rev{In the main implementation summary, ConQuR, MMUPHin, PLSDA-batch, DEBIAS-M and MetaDICT are grouped as microbiome-oriented methods, whereas ComBat, limma, ComBat-seq and FAbatch are grouped as general-purpose or expression-derived adjustment frameworks. The broader server method set is further mapped by input scale and model family in Supplementary Note 1.}


The assessment module applies the same metrics before and after correction, allowing direct method comparison under a common framework. For compositional analyses, abundances are transformed using the centered log-ratio \rev{(CLR)} transformation; \rev{Aitchison distance is the Euclidean distance between CLR-transformed profiles, whereas Bray--Curtis dissimilarity is computed on relative-abundance profiles.} Diagnostics include PCA, PCoA and NMDS in Aitchison and Bray--Curtis spaces, ANOSIM \citep{Clarke1993ANOSIM} and PERMANOVA \citep{Anderson2001PERMANOVA}, feature-wise ANOVA, pRDA, PVCA and neighborhood-based metrics such as alignment, entropy of batch mixing and silhouette summaries. \rev{Ordination plots summarize dominant sample structure and show whether points group by batch or by target. ANOSIM and PERMANOVA test distance-based batch association and report effect-size statistics. Feature-wise ANOVA, pRDA and PVCA summarize how variation is apportioned to batch, target, their overlap and residual components. Alignment and entropy of batch mixing describe local cross-batch mixing, whereas silhouette summaries describe separation with respect to the target labels.} The server also provides an example dataset and contextual help to facilitate rapid trial runs and interpretation.
Full descriptions of these analyses and metrics, \rev{including category mappings for correction methods and diagnostic outputs}, are provided in Supplementary Note 1.


\section{Results}
\subsection{Illustrative case study}
We demonstrated mBatchNet using the anaerobic digestion 16S rRNA dataset from a phenol-perturbation experiment, following the benchmark setup of PLSDA-batch~\citep{Chapleur2016Phenol,Wang2023PLSDABatch}. This example is bundled in the server so that users can reproduce the analysis directly from the web interface. The dataset comprises 231 OTUs measured across 75 samples. The \rev{target variable} was initial phenol concentration (0--0.5 versus 1--2), the \rev{batch variable} represented five processing dates, and treatment duration was included as an optional covariate where supported. \rev{The two target groups contained 26 and 49 samples, and batch sizes ranged from 9 to 21 samples with non-uniform target composition across batches.} After data upload, mBatchNet generates a mosaic plot summarizing the joint distribution of samples across batch and \rev{target} groups. The resulting plot for this dataset is shown in Supplementary Figure~S1.

Pre-correction diagnostics showed clear batch-associated structure (Figure~\ref{fig1}). In Aitchison ordinations, the uncorrected data clustered by processing date (Figure~\ref{fig1}A), and Bray--Curtis distance-based analyses indicated strong batch association (Figure~\ref{fig1}B; ANOSIM $R=0.637$, \rev{$p=0.001$}; PERMANOVA \rev{$R^2=0.457$, $p=0.001$}). After correction, mBatchNet revealed clear method-dependent differences. ConQuR, MMUPHin, DEBIAS-M, and MetaDICT increased overlap across batches while maintaining phenotype separation, whereas PLSDA-batch preserved phenotype-associated structure but retained more visible residual batch separation (Figure~\ref{fig1}A,B; Supplementary Figures~S2--S6). \rev{Among the displayed corrected outputs, ConQuR, DEBIAS-M and MMUPHin had Bray--Curtis ANOSIM values close to zero ($R=-0.052$ to $-0.010$, $p=0.648$--$0.981$) and PERMANOVA values of $R^2=0.029$--$0.069$ ($p=0.148$--$0.994$). MetaDICT had ANOSIM $R=0.039$ ($p=0.109$) and PERMANOVA $R^2=0.093$ ($p=0.009$), whereas PLSDA-batch had ANOSIM $R=0.377$ ($p=0.001$) and PERMANOVA $R^2=0.377$ ($p=0.001$).} Variance-partitioning analyses, including feature-wise ANOVA $R^2$ (Figure~\ref{fig1}C), pRDA (Figure~\ref{fig1}D), and PVCA (Supplementary Figure~S7), together with mixing and separation metrics (Supplementary Figure~S8), were consistent with these patterns. \rev{Together, the ordination, distance-based, variance-partitioning and neighborhood metrics describe complementary aspects of the same comparison: residual batch separation, phenotype separation, variance attribution and local cross-batch mixing.} All generated figures, corrected tables, and run outputs can be downloaded directly from the server for downstream analysis and reporting.

This case study illustrates the main role of mBatchNet, namely to provide a harmonized environment in which users can inspect study design, compare multiple correction outputs, and interpret correction trade-offs using consistent visual and statistical evidence.

\section{Conclusion}
mBatchNet provides an accessible and reproducible web-based framework for diagnosing, correcting and benchmarking batch effects in microbiome data. By integrating a broad set of recent correction methods with harmonized pre- and post-correction diagnostics, example data and downloadable outputs, the server enables transparent comparison of correction strategies and supports more reliable microbiome data integration and downstream analysis.
